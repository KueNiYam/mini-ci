Mini CI Product Requirements

Mini CI provides a small HTTP-triggered CI server for directories that are already deployed on the CI machine.

Mini CI는 CI 머신에 이미 배포된 디렉터리를 대상으로 동작하는 작은 HTTP trigger 기반 CI 서버입니다.

Goal

Let a user deploy source files with any tool, then run configured shell commands from the Mini CI dashboard or API.

사용자가 원하는 도구로 소스 파일을 배포한 뒤, Mini CI 대시보드나 API에서 설정된 shell command를 실행할 수 있게 합니다.

Core Workflow

deploy files to CI machine
-> POST /api/projects/:name/runs
-> run commands in the registered directory
-> store status and logs
-> view result in dashboard

Interfaces

Projects are registered with mini-ci project add <name> --path <directory> --cmd <command>.

프로젝트는 mini-ci project add <name> --path <directory> --cmd <command>로 등록합니다.

Runs are triggered with POST /api/projects/:name/runs and a bearer trigger token.

run은 bearer trigger token을 포함한 POST /api/projects/:name/runs 요청으로 시작합니다.

Security

The admin page requires MINI_CI_ADMIN_TOKEN to create or replace the trigger token.

admin 페이지는 trigger token을 생성하거나 교체할 때 MINI_CI_ADMIN_TOKEN을 요구합니다.

The trigger token is stored as a hash and is shown only when created or replaced.

trigger token은 hash로 저장되며, 생성하거나 교체한 직후에만 원문을 표시합니다.
