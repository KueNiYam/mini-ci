Mini CI Wiki

Mini CI runs checks against a directory that has already been deployed to the CI machine.

Mini CI는 CI 머신에 이미 배포된 디렉터리를 대상으로 검증 command를 실행합니다.

Concept

The deployment step is outside Mini CI. After deployment finishes, call Mini CI over HTTP to create a run.

배포 단계는 Mini CI 밖에서 처리합니다. 배포가 끝난 뒤 HTTP로 Mini CI를 호출해 run을 생성합니다.

This keeps copying, syncing, source selection, and release promotion policies separate from CI execution.

파일 복사, 동기화, 소스 선택, release promotion 정책을 CI 실행과 분리합니다.

Start The Server

Set an admin token before exposing the server on a LAN address.

LAN 주소로 서버를 열기 전 admin token을 설정합니다.
MINI_CI_ADMIN_TOKEN=admin-secret \
bin/mini-ci start --host 0.0.0.0 --port 4177

Register A Project

Register the directory where commands should run. The directory must already exist on the Mini CI machine.

command를 실행할 디렉터리를 등록합니다. 해당 디렉터리는 Mini CI 머신에 이미 존재해야 합니다.
bin/mini-ci project add storyboard \
  --path /Users/kueni-16/ci-sources/storyboard/current \
  --cmd "test -f README.md"

Manage Trigger Token

Open /admin, enter the admin token, and generate or save a trigger token.

/admin을 열고 admin token을 입력한 뒤 trigger token을 생성하거나 저장합니다.

The trigger token is stored as a hash. The plain token is shown only when it is created or replaced.

trigger token은 hash로 저장됩니다. 원문 token은 생성하거나 교체한 직후에만 표시됩니다.

Trigger A Run

After deployment finishes, call the run API with the trigger token.

배포가 끝난 뒤 trigger token을 포함해 run API를 호출합니다.
curl -X POST http://kueni-16.local:4177/api/projects/storyboard/runs \
  -H "Authorization: Bearer $MINI_CI_TRIGGER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"ref":"manual-20260511"}'

Dashboard

The dashboard shows all projects by default and can filter jobs by project.

대시보드는 기본적으로 전체 프로젝트를 보여주며, 프로젝트별 job 필터링을 지원합니다.

Job details show project, status, ref, failed step, exit code, creation time, and logs.

job 상세에는 프로젝트, 상태, ref, 실패 step, exit code, 생성 시각, 로그가 표시됩니다.

Troubleshooting

  • 409 trigger token is not configured: create a trigger token from /admin.
  • 401 invalid trigger token: update the caller to use the current trigger token.
  • 404 project not found: check the project name passed to project add.
  • Command failures are recorded in the job log and keep the job status as failed.
문제가 생기면 token 설정, 프로젝트 이름, job 로그의 실패 command를 먼저 확인합니다.
