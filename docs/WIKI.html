Mini CI Wiki

Mini CI runs checks against a directory that has already been deployed to the CI machine.

Mini CI는 CI 머신에 이미 배포된 디렉터리를 대상으로 검증 command를 실행합니다.

Concept

The deployment step is outside Mini CI. After deployment finishes, call Mini CI over HTTP to create a run.

배포 단계는 Mini CI 밖에서 처리합니다. 배포가 끝난 뒤 HTTP로 Mini CI를 호출해 run을 생성합니다.

This keeps copying, syncing, source selection, and release promotion policies separate from CI execution.

파일 복사, 동기화, 소스 선택, release promotion 정책을 CI 실행과 분리합니다.

Start The Server

Start the server on the CI machine. If you expose it on a LAN address, anyone who can reach it can register projects and trigger runs.

CI 머신에서 서버를 시작합니다. LAN 주소로 열면 접근 가능한 사용자는 누구나 프로젝트 등록과 run 실행을 할 수 있습니다.

Project paths are discovered by project name under ~/.codex/worktrees by default. Set MINI_CI_PROJECT_ROOT when another base directory is needed.

프로젝트 경로는 기본적으로 ~/.codex/worktrees 아래에서 프로젝트명으로 자동 탐지합니다. 다른 기준 디렉터리가 필요하면 MINI_CI_PROJECT_ROOT를 설정합니다.
MINI_CI_PORT="<port>"
bin/mini-ci start --host 0.0.0.0 --port "${MINI_CI_PORT}"

Register A Project

Open /admin, choose a project, and save it once. The client deployment process should refresh the project and start a run after it prepares the worktree directories.

/admin을 열고 프로젝트를 선택해 한 번 저장합니다. 이후 클라이언트 배포 프로세스는 worktree 디렉터리를 준비한 뒤 프로젝트를 갱신하고 run을 시작해야 합니다.

Use these calls from the client process after deployment finishes.

배포가 끝난 뒤 클라이언트 프로세스에서 아래 호출을 사용합니다.

WORKTREE_PATH selects the worktree to run. RUN_DATE becomes the date row on the dashboard.

WORKTREE_PATH는 실행할 worktree를 선택합니다. RUN_DATE는 대시보드의 날짜 행으로 표시됩니다.
MINI_CI_URL="http://<mini-ci-host>:<port>"
PROJECT_NAME="<project-name>"
WORKTREE_ID="<worktree-id>"
WORKTREE_PATH="${WORKTREE_ID}/${PROJECT_NAME}"
RUN_DATE="$(date +%Y%m%d%H%M%S)"
COMMANDS='["npm test"]'

curl -X POST "${MINI_CI_URL}/api/admin/projects" \
  -H "Content-Type: application/json" \
  -d "{\"name\":\"${PROJECT_NAME}\",\"paths\":[\"${WORKTREE_PATH}\"],\"commands\":${COMMANDS}}"

curl -X POST "${MINI_CI_URL}/api/projects/${PROJECT_NAME}/runs" \
  -H "Content-Type: application/json" \
  -d "{\"worktreePath\":\"${WORKTREE_PATH}\",\"runDate\":\"${RUN_DATE}\"}"

Dashboard

The dashboard groups history by project name and worktree ID. Each run appears as a date row.

대시보드는 실행 이력을 project name과 worktree ID 기준으로 묶고, 각 run을 날짜 행으로 표시합니다.

Job details show project, worktree ID, worktree path, run date, status, failed step, exit code, creation time, and logs.

job 상세에는 프로젝트, worktree ID, worktree path, run date, 상태, 실패 step, exit code, 생성 시각, 로그가 표시됩니다.

Troubleshooting

  • 404 project not found: check the project name registered from /admin.
  • Command failures are recorded in the job log and keep the job status as failed.
문제가 생기면 프로젝트 이름과 job 로그의 실패 command를 먼저 확인합니다.
