# Mini CI 프로젝트 규칙

- `README.md`처럼 짧은 공개/온보딩 문서는 영어를 기본 본문으로 작성하고, 한국어 번역은 바로 아래 blockquote로 제공합니다.
- `docs/` 아래 긴 사용자용 문서는 언어별 HTML 파일로 분리할 수 있습니다. 기본 영어 문서는 `*.html`, 한국어 문서는 `*.ko.html`로 둡니다.
- 언어별 문서는 가능한 한 같은 섹션 구조와 같은 anchor를 유지하고, 각 문서 상단에 언어 전환 링크를 둡니다.
- 언어별로 분리한 문서 안에서는 영어/한국어 blockquote 병기를 반복하지 않습니다.
- `docs/` 아래 사용자용 문서는 기본적으로 HTML 파일로 작성하며, 공통 스타일 같은 지원 asset은 필요할 때 별도 CSS로 둘 수 있습니다.
- 새로 작성하거나 수정하는 코드 주석과 TSDoc은 병기하지 않고 한국어로 작성합니다.
- `TODO.md`처럼 내부 작업 추적용 문서는 병기하지 않아도 됩니다.
- 제목, 명령어, API 경로, 파일명, 코드 식별자처럼 그대로 실행하거나 참조해야 하는 값은 번역하지 않고 영어 원문을 유지합니다.
- 짧은 용어형 항목은 필요할 때 `English / 한국어` 형태를 사용할 수 있습니다.
